# RDP & AD CRM Моніторинг

Комплексна система для моніторингу підключень RDP та журналювання подій безпеки Active Directory (AD). Вона поєднує в собі легкі фонові агенти збору даних (написані на Python) та сучасну веб-панель аналітики з веденням обліку робочого часу, розрахунком понаднормових годин та Cupertino Light Theme дизайном.

---

## 📁 Структура проекту

* **[CRM-WEB](file:///j:/Program%20Files/rdp_monitor/CRM-WEB)** — Веб-інтерфейс управління, аналітики та API-ендпоінти для прийому подій (PHP + MySQL).
* **[monitoring_agents](file:///j:/Program%20Files/rdp_monitor/monitoring_agents)** — Клієнтські скрипти збору подій на базі Python, які запускаються безпосередньо на серверах.
  * **[rdp_monitor](file:///j:/Program%20Files/rdp_monitor/monitoring_agents/rdp_monitor)** — Відстеження RDP-підключень, тривалості сесій та активності користувачів.
  * **[ad_monitor](file:///j:/Program%20Files/rdp_monitor/monitoring_agents/ad_monitor)** — Аудит контролера домену Active Directory (створення, видалення, блокування користувачів, зміна паролів).
* **[.gitignore](file:///j:/Program%20Files/rdp_monitor/.gitignore)** — Конфігурація виключень Git (ігноруються `.exe`, локальні бази даних, файли конфігурації з паролями та токенами).

## 🖥️ CRM-WEB (Веб-панель управління)

### Системні вимоги:
* Веб-сервер (Apache, Nginx або IIS)
* PHP версії 8.1 або новішої
* СУБД MySQL / MariaDB

### Інструкція зі встановлення на хостинг:
1. Завантажте весь вміст директорії `CRM-WEB` на ваш веб-хостинг (наприклад, у папку сайту `/home/apgk/web/rdp.dniprovska.net/public_html/`).
2. Створіть нову базу даних MySQL на хостингу та користувача бази з повними правами доступу.
3. Відкрийте файл конфігурації підключення до БД [CRM-WEB/includes/db.php](file:///j:/Program%20Files/rdp_monitor/CRM-WEB/includes/db.php) та пропишіть ваші реквізити:
   ```php
   define('DB_HOST', 'localhost');
   define('DB_NAME', 'назва_вашої_бази');
   define('DB_USER', 'ім_я_користувача_бази');
   define('DB_PASS', 'пароль_користувача_бази');
   ```
4. Відкрийте ваш сайт у браузері на сторінці ініціалізації таблиць (наприклад, `https://rdp.dniprovska.net/install.php`). Скрипт автоматично створить схему бази даних та запише супер-адміністратора за замовчуванням:
   * **Логін:** `admin`
   * **Пароль:** `admin`
5. **ВАЖЛИВО:** Після успішного налаштування обов'язково видаліть або перейменуйте файл `install.php` на хостингу з міркувань безпеки!
6. Увійдіть у панель, перейдіть до розділу **«Токени API»** та згенеруйте API-токени для служб збору даних.

## 🐍 monitoring_agents (Клієнтські агенти)

Фонові скрипти, що працюють безпосередньо на Windows-серверах (термінальних серверах та контролерах домену).

### Загальні вимоги:
* Python версії 3.8 або новішої
* Встановлені залежності (запускати через cmd/PowerShell з правами адміністратора):
  ```bash
  pip install pywin32 requests
  ```

---

### 1. RDP Монітор (`monitoring_agents/rdp_monitor`)
Моніторить сеанси RDP на термінальних серверах і передає дані в CRM та Telegram.

* **Налаштування:**
  Створіть у папці скрипта конфігураційний файл `rdp_monitor_config.json`:
  ```json
  {
      "server_name": "Назва_Вашого_RDP_Сервера",
      "bot_token": "Токен_Вашого_Telegram_Бота_Для_Сповіщень",
      "telegram_chats": [
          {
              "active": true,
              "name": "Назва_Чату",
              "id": "ID_Чату_З_Мінусом"
          }
      ],
      "db_type": "sqlite",
      "pg_settings": {}
  }
  ```
  *Якщо використовується синхронізація з хмарою Google, також покладіть туди файл доступу `google_creds.json`.*

* **Запуск:**
  Для постійної роботи скрипт можна запустити у фоні або налаштувати запуск при завантаженні системи через Планувальник завдань Windows (Task Scheduler).

---

### 2. AD Монітор (`monitoring_agents/ad_monitor`)
Аналізує події журналу безпеки Windows (Security Event Log) на контролері домену.

* **Налаштування:**
  Створіть конфігураційний файл `ad_monitor_config.json` у папці скрипта:
  ```json
  {
      "dc_name": "Назва_Контролера_Домену",
      "crm_api_url": "https://rdp.dniprovska.net/api/ad_event.php",
      "crm_api_token": "Згенерований_Токен_З_Веб_Панелі"
  }
  ```

* **Запуск:**
  Запускається як фоновий процес на контролері домену.

### 📦 Збирання в автономний виконуваний файл (`.exe`)
Щоб запускати монітори на серверах без необхідності встановлення Python, їх можна скомпілювати:

1. Встановіть інструмент компіляції:
   ```bash
   pip install pyinstaller
   ```
2. Зберіть RDP Монітор:
   ```bash
   cd monitoring_agents/rdp_monitor
   pyinstaller rdp_monitor.spec
   ```
3. Зберіть AD Монітор:
   ```bash
   cd monitoring_agents/ad_monitor
   pyinstaller AD_monitor.spec
   ```
4. Скомпільовані програми з'являться в каталогах `dist/` відповідних папок. Перенесіть `.exe` файли та відповідні конфігураційні JSON на цільові сервери.
